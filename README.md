# 公開講座運営アシスタント Ver.1.1

病院広報担当者向けに、公開講座の運営で使う文章と Google Apps Script を生成する静的Webツールです。

講座情報を入力すると、以下をブラウザ内だけで生成できます。

- 自動返信メール用 Apps Script
- リマインダーメール用 Apps Script
- 自動返信メール本文プレビュー
- リマインダーメール本文プレビュー
- Googleフォーム説明文
- Googleフォーム送信完了画面文
- ホームページ掲載用の注意文

外部ライブラリ、外部API、サーバー通信は使っていません。入力された個人情報や講座情報は外部へ送信されません。

## Ver.1.1 の主な改善点

- 基本設定エリアを折りたたみ式にしました。
- 基本設定をブラウザ内の `localStorage` に保存できるようにしました。
- 保存済みの基本設定は、次回ページを開いたときに自動反映されます。
- 開催日を選択すると、曜日が自動入力されます。
- 会場テンプレートを講座情報より上に移動し、先に会場を選びやすくしました。

## 使い方

1. `index.html` をブラウザで開きます。
2. 必要に応じて「基本設定を開く」を押し、病院情報を確認・編集します。
3. 基本設定を変更した場合は「基本設定を保存」を押します。
4. 必要に応じて「よく使う会場テンプレート」を押し、会場情報を自動入力します。
5. 講座情報を入力します。開催日を選ぶと曜日が自動入力されます。
6. 「コードを生成」を押します。
7. 表示された各ボックスの「コピー」ボタンから、必要な文章やコードをコピーします。

「サンプルを入力」を押すと、動作確認用の講座情報が入ります。
「入力をクリア」を押すと、講座情報だけを初期状態に戻します。
「基本設定を初期値に戻す」を押すと、保存済みの基本設定を削除し、初期値に戻します。

## Google Apps Script への貼り付け方法

1. Googleフォームの回答先スプレッドシートを開きます。
2. メニューから「拡張機能」→「Apps Script」を開きます。
3. 生成された「自動返信メール用 Apps Script」を貼り付けます。
4. リマインダーメールも使う場合は、別ファイルまたは同じプロジェクト内に「リマインダーメール用 Apps Script」を貼り付けます。
5. 保存後、必要なトリガーを設定します。

## トリガー設定方法

### autoReply

- 実行する関数：`autoReply`
- イベントのソース：スプレッドシートから
- イベントの種類：フォーム送信時

### sendReminder

- 実行する関数：`sendReminder`
- イベントのソース：時間主導型
- 時間ベースのトリガーのタイプ：日付ベースのタイマー
- 時刻：午前9時〜10時

`sendReminder` は、開催日の3日前から当日まで毎朝チェックし、まだリマインド未送信の申込者だけに送信します。

## 注意事項

- B列がメールアドレス、C列が名前である前提です。
- `申込返信済み`、`返信日時`、`リマインド送信済み`、`リマインド日時` の列は削除しないでください。
- テスト時は、まず自分のメールアドレスで送信確認してください。
- Google Apps Script 初回実行時は、Googleの権限承認が必要です。
- 入力値はそのままメール本文や案内文に反映されます。公開前に内容を確認してください。
- このツール自体はサーバー通信を行わず、個人情報を外部送信しない設計です。
- 基本設定の保存にはブラウザの `localStorage` を使用します。同じPC・同じブラウザで利用する前提です。
